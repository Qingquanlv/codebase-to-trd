## Template Usage Rules

1. **Replace all `{{...}}` placeholders** with actual content from the analysis
2. **TOC entries** must exactly match the sections in the content — add/remove entries for modules as needed
3. **Mermaid diagrams** are placed inside `<div class="diagram-container"><div class="mermaid">...</div></div>`
4. **Code blocks** use `<pre><code class="language-xxx">...</code></pre>`
5. **Module cards** wrap each 3.2.x module for visual separation
6. **Evaluation grid** uses the two-column layout for 优势 vs 局限
7. Section IDs follow the pattern: `section-1`, `section-1-1`, `section-1-2-1`, etc.
8. Mermaid and Highlight.js are loaded from CDN — the HTML is fully self-contained otherwise
9. Agent should **add or remove** 预期/理念/模块 subsections based on what the analysis actually finds — the template shows the minimum structure, not a rigid count
